根据项目的 `CONTRIBUTING.md`，该项目的开发流程如下：

## 正常的开发流程

### 1. Fork 和克隆仓库
```bash
# 首先在GitHub上Fork这个仓库
# 然后克隆你Fork的仓库到本地
git clone https://github.com/你的用户名/marquez-airflow.git
cd marquez-airflow
```

### 2. 配置上游仓库（可选但推荐）
```bash
# 添加上游仓库，方便后续同步更新
git remote add upstream https://github.com/MarquezProject/marquez.git
```

### 3. 创建新分支
根据修改类型创建分支：
```bash
# 添加新功能
git checkout -b feature/你的功能名称

# 修复bug
git checkout -b bug/你的bug修复

# 提出提案
git checkout -b proposal/你的提案
```

分支命名规则：
- 使用小写字母
- 用短横线 `-` 分隔单词
- 使用描述性的名称

### 4. 进行代码修改
进行你的修改。

### 5. 运行测试
```bash
# 运行所有测试
./gradlew test

# 运行特定模块的测试
./gradlew :api:test

# 运行单元测试
./gradlew :api:testUnit
```

### 6. 格式化代码
项目使用 spotless 格式化代码：
```bash
# 自动格式化代码
./gradlew spotlessApply

# 检查代码格式
./gradlew spotlessJavaCheck
```

### 7. 安装 Git Hooks（推荐）
```bash
# 安装pre-commit
brew install pre-commit

# 设置git hooks
pre-commit install
```

### 8. 提交代码
提交时需要签名（Sign-off）：
```bash
# 添加修改的文件
git add .

# 提交（使用 -s 参数自动添加签名）
git commit -s -m "你的提交信息"

# 或者手动添加签名
git commit -m "你的提交信息

Signed-off-by: 你的名字 <your.email@example.com>"
```

提交信息要求：
- 清晰描述修改内容
- 如果修复了bug，在PR描述中链接相关issue

### 9. 推送到你的Fork
```bash
git push origin feature/你的功能名称
```

### 10. 创建 Pull Request
1. 在 GitHub 上打开你的 Fork
2. 点击 "New Pull Request"
3. 选择你的分支
4. 填写 PR 描述，说明修改内容
5. 提交 PR

### 11. 代码审查和合并
- 等待项目维护者审查
- 根据反馈进行修改
- 维护者批准后合并到主分支

## 重要注意事项

1. 代码格式：确保运行 `./gradlew spotlessApply`，否则 CI 会失败
2. 测试：所有修改都应包含测试
3. 版权声明：Java、Bash、Python 文件需要包含版权和许可证头
4. 提交签名：所有提交都需要 Sign-off
5. 单一提交：一个 PR 通常只包含一个提交（可以使用 `git commit --amend` 或 squash）

## 额外提示

- 保持 PR 的 diff 小而独立
- 如果修改了 API，需要重新生成 `openapi.html`
- 可以加入项目的 Slack 频道获取帮助

## 合并到 GitHub 主干的步骤

### 情况 1：合并到自己的 Fork 仓库的 main 分支

如果当前在功能分支（如 `feature/optimize`），需要合并到自己的 `main` 分支：

```bash
# 1. 确保所有修改已提交
git status

# 2. 切换到 main 分支
git checkout main

# 3. 拉取最新的 main 分支（如果有远程更新）
git pull origin main

# 4. 合并功能分支到 main
git merge feature/你的分支名

# 5. 推送到 GitHub
git push origin main
```

### 情况 2：合并到上游仓库（MarquezProject/marquez）的 main 分支

如果要合并到上游仓库，需要通过 Pull Request：

```bash
# 1. 确保当前分支的所有修改已提交
git status

# 2. 检查当前分支名称
git branch

# 3. 推送到你的 Fork 仓库
git push origin feature/你的分支名

# 4. 在 GitHub 上创建 Pull Request
#    - 访问 https://github.com/你的用户名/marquez-airflow
#    - 点击 "New Pull Request"
#    - 选择 base: main <- compare: feature/你的分支名
#    - 填写 PR 描述
#    - 提交 PR
```

### 情况 3：直接推送到 main 分支（如果已有权限）

如果这是你自己的仓库，可以直接推送：

```bash
# 1. 确保所有修改已提交
git add .
git commit -s -m "feat: Web UI 优化和部署文档更新

- 优化 Web UI 文本显示和血缘图谱
- 添加6层复杂数据血缘测试脚本
- 完善部署文档和使用指南
- 添加数据管理脚本说明

Signed-off-by: 你的名字 <your.email@example.com>"

# 2. 推送到 main 分支
git push origin main
```

### 重要提示

1. 提交前检查：
   ```bash
   # 检查代码格式
   ./gradlew spotlessApply
   
   # 运行测试
   ./gradlew test
   ```

2. 提交信息格式：
   - 使用 `-s` 参数自动添加 Sign-off
   - 提交信息要清晰描述修改内容

3. 如果遇到冲突：
   ```bash
   # 拉取最新代码
   git pull origin main
   
   # 解决冲突后
   git add .
   git commit -s -m "解决合并冲突"
   git push origin main
   ```

